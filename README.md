## Описание проекта
Это API для управления событиями с регистрацией пользователей, аутентификацией и возможностью бронирования мест на события. Реализовано с использованием Django REST Framework.

---

## Настройка и запуск
1. Все переменные окружения можно настроить в файле `.env`.
2. Для запуска проекта используйте команду: `docker-compose up -d`
3. Для остановки и удаления контейнеров с томами выполните: `docker-compose down -v`

---

## Доступные эндпоинты API
### Аутентификация
- **POST** `/auth/register/`  
Регистрирует пользователя. Принимает:
```
{
"username": "string",
"password": "string"
}
```
Возвращает токен авторизации.

- **POST** `/auth/login/`  
Логин пользователя. Принимает:
```
{
"username": "string",
"password": "string"
}
```
Возвращает токен авторизации.

- **POST** `/auth/logout/`  
Выход из системы для авторизованного пользователя.

---

### События
- **GET** `/events/`  
Получение списка всех событий.

- **POST** `/events/`  
Создание нового события. Принимает JSON в формате:
```
{
"title": "Концерт",
"tags": ["tag1", "tag2", "tag4"],
"description": "Описание концерта",
"start_time": "2025-05-10T19:00:00Z",
"location": "Москва",
"seats": 1,
"status": 1
}
```

- **GET** `/events/{pk}/`  
Получение информации по конкретному событию.

- **PUT** `/events/{pk}/`  
Обновление события (формат запроса аналогичен созданию).

- **DELETE** `/events/{pk}/`  
Удаление события.

---

### Бронирование

- **POST** `/events/{pk}/create_booking/`  
Бронирование места пользователем на событие.

- **DELETE** `/events/{pk}/delete_booking/`  
Отмена бронирования пользователя на событие.

---

## Дополнительно

- Для работы с API требуется авторизация через токен.
- Все операции с событиями и бронированиями защищены и требуют соответствующих прав пользователя.
- Все GET запросы доступны без авторизации.
- Уведомления могут потеряться, желательно реализовать transactional outbox.
